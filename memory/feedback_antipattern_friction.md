name: Anti-pattern friction → ask user
description: When a skeleton's anti-patterns contradict a practical design decision, stop and ask the user instead of agonizing
type: feedback
---

When an anti-pattern in a task skeleton creates friction with a practical design call, STOP and ask the user. Don't spend time analyzing the tension — surface it.

**Why:** Agents keep writing anti-patterns that become contradiction factories. Anti-patterns are guidelines captured at planning time; practical design decisions happen at implementation time with fuller context. The anti-pattern author didn't foresee the specific trade-off.

**How to apply:** If you're debating "the anti-pattern says X but the practical answer is Y" for more than one paragraph, that's the signal. Present both sides in 2-3 sentences and ask.
